# UI Smoke Checklist (Admin Panel)

Пройти вручную перед merge (если e2e не запускались):

1. `GET /health/live`, `GET /health/ready`.
2. Первый вход:
   - при отсутствии admin доступен `/setup`,
   - после создания admin `/setup` больше не создаёт нового admin.
3. `login`/`logout`:
   - вход по корректному паролю,
   - неверный пароль возвращает нейтральную ошибку.
4. `forgot/reset`:
   - форма отправляется,
   - в dev-режиме token виден только при `SHOW_DEV_TOKENS=1`,
   - reset инвалидирует старые сессии.
5. Онбординг:
   - страница `/onboarding` открывается для admin,
   - заголовок и подсказки человеко-понятные (Redmine/Matrix + примеры),
   - `Пропустить` включает warning-баннер,
   - сохранение секретов снимает warning.
6. Сайдбар: **Дашборд** → **Группы** → **Пользователи** → **Настройки**; внизу (admin) — **События**. **Привязка Matrix** — в меню пользователя в шапке. **Аккаунты панели** — в том же меню для admin.
7. Пользователи:
   - есть поля `Имя`/`Отдел`,
   - работает поиск и фильтр по отделу,
   - есть группировка по отделам со сворачиванием.
8. Мои настройки (`/me/settings`, пункт меню пользователя — не admin):
   - нет обязательного JSON-ввода,
   - работают пресеты уведомлений + расширенный режим,
   - рабочие дни задаются чекбоксами, часы через диапазон `с/до`.
   - учётка **admin** с `/me/settings` уходит на дашборд (`/`).
9. Маршруты по статусу:
   - настраиваются в форме **группы** (после сохранения комнаты): список, добавление по одному, удаление;
   - при смене `room_id` группы строки `status_room_routes` с прежней комнатой переносятся на новую (осторожно при общей комнате у двух групп);
   - `GET /routes/status` редиректит на `/groups`.
10. Matrix binding:
   - экран и тексты в едином стиле панели,
   - понятный двухшаговый сценарий отправки/подтверждения кода.
11. Секреты:
   - в списке только метаданные, без значений.
12. Адаптивность:
   - базовые страницы читаемы на ширине 1280 и 375px;
   - на узкой ширине сайдбар открывается кнопкой, overlay закрывает меню.
13. **CSRF:** все изменяющие состояние POST-формы отправляют `csrf_token` (скрытое поле); для HTMX заголовок `X-CSRF-Token` подставляется из `meta` в `base.html`.
14. **Маршруты по версии (`/routes/version`):** доступ с дашборда (карточка-счётчик); длинные Matrix `room_id` в таблице обрезаются с подсказкой по `title`.
15. **Меню пользователя (шапка):** при открытии фокус на первом пункте; `↑`/`↓` с циклическим переходом, `Home`/`End`; `Escape` закрывает меню и возвращает фокус на кнопку (сайдбар на мобильных закрывается отдельным `Escape`, если меню закрыто).
16. **Формы:** вертикальный ритм через `.form` / `.field`; страницы входа и setup/forgot/reset в том же духе, что и панель (без «пляшущих» отступов у первого поля).
17. **Заголовки и таблицы:** единая шапка страниц; lead только там, где нужен по смыслу; в таблицах колонка действий выровнена, пустое состояние читаемо.
18. **Статика и CSP:** `GET /static/admin/css/panel.css` отдаётся; при включённом `ADMIN_ENABLE_CSP` страницы получают заголовок `Content-Security-Policy` (проверить в DevTools → Network).
19. **Версия CSS:** в HTML ссылки вида `/static/admin/css/panel.css?v=…` и `auth.css?v=…` (значение из `ADMIN_ASSET_VERSION`, по умолчанию `1`); после смены стилей в деплое увеличить версию в `.env`, чтобы браузер подтянул новый файл без принудительного обновления.
20. **Баннер подключений** (если секреты не заполнены): полная полоса сворачивается кнопкой в компактную ссылку «Настройки»; после перезагрузки состояние сохраняется (`localStorage`).
21. **Автотесты E2E (опционально):** при `DATABASE_URL` и `playwright install chromium` — `pytest tests/e2e/`; при уже созданном админе задать `E2E_ADMIN_LOGIN` / `E2E_ADMIN_PASSWORD` или прогонять на пустой БД после миграций.
22. **Группы и пользователи:** список групп без служебной `UNASSIGNED`; в форме пользователя — «обычные» группы и «Без группы»; фильтр на `/users` без дубля `UNASSIGNED` и без дубля подписи «Все группы»; подстановка имени из Redmine по user ID.
23. **События и дашборд:** на `/events` нет длинных подсказок про путь к логу (детали в README / `ADMINISTRATOR_GUIDE`); на дашборде нет абзаца про Postgres и перезапуск бота (то же в документации). Карточка «Маршруты по версии» ведёт на `/routes/version`.
