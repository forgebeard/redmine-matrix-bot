# UI Smoke Checklist (Admin Panel)

## Legacy routing decisions (Act 3)

Решения зафиксированы по коду и репозиторию (нет ссылок `action="/routes/status…"` и нет ссылок на `/audit` вне удалённых тестов). Логи `legacy_route_used` на проде при расхождении — пересмотреть строки ниже.

| URL | Решение | Обоснование |
|-----|---------|-------------|
| `GET /audit` | **Удалить** | Журнал — `/events`; внешних потребителей в репозитории не найдено. |
| `GET /audit/export.csv` | **Удалить** | Выгрузка — `/events/export.csv`; см. выше. |
| `GET /routes/status` | **Оставить GET-редирект** на `/groups` | Закладки на старый URL. |
| `POST /routes/status`, `POST /routes/status/by-room`, `POST /routes/status/{id}/delete` | **Удалить** | Настройка — только из карточки группы (`/groups/{id}/status-routes/*`); форм на legacy POST в репозитории нет. |
| `GET /routes/version` | **301** на `/settings/routes/version` | Канонический путь глобальных маршрутов версий (Act 4). |
| `POST /routes/version`, `POST /routes/version/{id}/delete` | **410 Gone** | POST только на `/settings/routes/version` / `.../delete`. |

Пройти вручную перед merge (если e2e не запускались):

1. `GET /health/live`, `GET /health/ready`.
2. Первый вход:
   - при отсутствии admin доступен `/setup`,
   - после создания admin `/setup` больше не создаёт нового admin.
3. `login`/`logout`:
   - вход по корректному паролю,
   - неверный пароль возвращает нейтральную ошибку.
4. Сброс пароля: самообслуживания по `/forgot-password` нет (редирект на `/login`); смена по токену — `/reset-password?token=…` при наличии токена в БД.
5. Онбординг:
   - страница `/onboarding` открывается для admin,
   - заголовок и подсказки человеко-понятные (Redmine/Matrix + примеры),
   - сохранение секретов снимает предупреждение в шапке (баннер «Подключения не настроены»).
6. Сайдбар: **Дашборд** → **Группы** → **Пользователи** → **Настройки подключений**; далее (admin) — **Аккаунты панели**, **События**. Дублирующих кнопок в баннере интеграции нет.
7. Пользователи:
   - есть поля `Имя`/`Отдел`,
   - работает поиск и фильтр по отделу,
   - есть группировка по отделам со сворачиванием.
8. Мои настройки (`/me/settings`, пункт меню пользователя — не admin):
   - нет обязательного JSON-ввода,
   - работают пресеты уведомлений + расширенный режим,
   - рабочие дни задаются чекбоксами, часы через диапазон `с/до`.
   - учётка **admin** с `/me/settings` уходит на дашборд (`/`).
9. Маршруты по статусу:
   - настраиваются в форме **группы** (после сохранения комнаты): список, добавление по одному, удаление;
   - при смене `room_id` группы строки `status_room_routes` с прежней комнатой переносятся на новую (осторожно при общей комнате у двух групп);
   - `GET /routes/status` (только редирект для закладок) ведёт на `/groups`; legacy POST к `/routes/status*` удалены — см. раздел «Legacy routing decisions» выше.
10. Секреты:
   - в списке только метаданные, без значений.
11. Адаптивность:
   - базовые страницы читаемы на ширине 1280 и 375px;
   - на узкой ширине сайдбар открывается кнопкой, overlay закрывает меню.
12. **CSRF:** все изменяющие состояние POST-формы отправляют `csrf_token` (скрытое поле); для HTMX заголовок `X-CSRF-Token` подставляется из `meta` в `base.html`.
13. **Маршруты по версии (`/settings/routes/version`):** с дашборда ссылка в блоке «Что сделать сейчас»; `GET /routes/version` отдаёт **301** на канонический путь; длинные Matrix `room_id` в таблице обрезаются с подсказкой по `title`.
14. **Меню пользователя (шапка):** при открытии фокус на первом пункте; `↑`/`↓` с циклическим переходом, `Home`/`End`; `Escape` закрывает меню и возвращает фокус на кнопку (сайдбар на мобильных закрывается отдельным `Escape`, если меню закрыто).
15. **Формы:** вертикальный ритм через `.form` / `.field`; страницы входа и setup/forgot/reset в том же духе, что и панель (без «пляшущих» отступов у первого поля).
16. **Заголовки и таблицы:** единая шапка страниц; lead только там, где нужен по смыслу; в таблицах колонка действий выровнена, пустое состояние читаемо.
17. **Статика и CSP:** `GET /static/admin/css/panel.css` отдаётся; при включённом `ADMIN_ENABLE_CSP` страницы получают заголовок `Content-Security-Policy` (проверить в DevTools → Network).
18. **Версия CSS:** в HTML ссылки вида `/static/admin/css/panel.css?v=…` и `auth.css?v=…` (значение из `ADMIN_ASSET_VERSION`, встроенный fallback в `admin_main` — см. код); после смены стилей в деплое увеличить версию в `.env`, чтобы браузер подтянул новый файл без принудительного обновления.
19. **Баннер подключений** (если секреты не заполнены): полная полоса сворачивается кнопкой в компактную ссылку «Настройки»; после перезагрузки состояние сохраняется (`localStorage`).
20. **Автотесты E2E (опционально):** при `DATABASE_URL` и `playwright install chromium` — `pytest tests/e2e/`; при уже созданном админе задать `E2E_ADMIN_LOGIN` / `E2E_ADMIN_PASSWORD` или прогонять на пустой БД после миграций.
21. **Группы и пользователи:** список групп без служебной `UNASSIGNED`; в форме пользователя — «обычные» группы и «Без группы»; фильтр на `/users` без дубля `UNASSIGNED` и без дубля подписи «Все группы»; подстановка имени из Redmine по user ID.
22. **События и дашборд:** на `/events` нет длинных подсказок про путь к логу (детали в README / `ADMINISTRATOR_GUIDE`); на дашборде нет абзаца про Postgres и перезапуск бота (то же в документации). Карточка «Пользователи» ведёт на `/users`, «Маршруты по статусу» — на `/groups`; глобальные маршруты версий — ссылка «Маршруты по версии (глобально)» на **`/settings/routes/version`**.
