# Руководство администратора

> Для развёртывания на сервере см. [DEPLOYMENT_GUIDE.md](DEPLOYMENT_GUIDE.md).
> Для обзора проекта см. [README.md](../README.md).
> Подробный сценарий первого запуска и связь «кнопка в админке → БД → бот»: [DAY_ZERO_EXTENDED.md](DAY_ZERO_EXTENDED.md).

## 1. Первый вход

### 1.1. Одноразовая регистрация (`/setup`)

Страница доступна, пока в БД нет ни одного администратора:

`http://<хост>:8080/setup`

Задайте логин и пароль. После этого используйте `/login`.

> Если `/setup` говорит что админ уже есть, а том `postgres_data` использовался ранее — запись могла остаться с прошлых запусков. Варианты: войти под существующим логином, сбросить пароль скриптом `scripts/reset_admin_password.py`, либо для чистой среды: `docker compose down -v` (данные БД удалятся).

### 1.2. Ограничение по логину

Переменная `ADMIN_LOGINS` в `.env` — список разрешённых логинов (через запятую). Пустое значение = без ограничения.

## 2. Настройка интеграций

После входа перейдите в **Настройки** (`/onboarding`):

1. **Параметры сервиса** — введите URL и API-ключ Redmine, параметры Matrix (Homeserver, Access Token, User ID). Нажмите **«Проверить доступ»**, затем **«Сохранить»**.
2. **База данных сервиса** — скопируйте `POSTGRES_PASSWORD` и `APP_MASTER_KEY` в безопасное место.
3. Перезапустите бота: `docker compose restart bot`.

**Matrix Device ID** (`MATRIX_DEVICE_ID`) в панели не задаётся: используйте переменную окружения или ключ `MATRIX_DEVICE_ID` в таблице `cycle_settings`; после смены перезапустите процесс бота.

**Шаблоны уведомлений Matrix:** вкладка «Уведомления» в Настройках (`/onboarding`). Справочник полей Jinja по каждому шаблону — [notification_template_variables.md](notification_template_variables.md). Тестовые отправки из панелей Пользователи/Группы используют только шаблон `tpl_test_message` (редактируется там же). Legacy-шаблон `tpl_dry_run` удалён из runtime/API.

### 2.1. Страница «Секреты» (`/secrets`)

Секреты лежат в таблице `app_secrets`. **Сохранение** — форма «Имя» и «Значение» (значение в интерфейс не выводится). **Удаление строки** — кнопка в таблице напротив имени; после удаления ключа бот при следующей загрузке конфигурации не увидит его в БД — снова введите значение в форме или задайте переменную в `.env` для подстраховки (см. развёртывание). Не удаляйте обязательные ключи без плана замены: интеграции перестанут работать до восстановления.

## 3. Панель: разделы

| Раздел | URL | Описание |
|--------|-----|----------|
| Дашборд | `/dashboard` | Плитки Пользователи / Группы / События, блок «Что сделать сейчас», управление ботом (Старт/Стоп/Рестарт) |
| Пользователи | `/users` | Список, создание, редактирование (Redmine ID, Matrix user, настройки) |
| Группы | `/groups` | Комната Matrix группы, маршруты по статусам Redmine, версии |
| Маршруты по версии | `/routes/version` | Глобальная карта «версия → комната Matrix» |
| Настройки | `/onboarding` | Параметры сервиса, БД, таймзона, справочник |
| Секреты | `/secrets` | Имена и метаданные ключей в БД; сохранение и удаление строк |
| Аккаунты панели | `/app-users` | Управление аккаунтами админки (смена логина, сброс пароля) |
| События | `/events` | Журнал событий бота и панели, фильтр по датам, CSV-экспорт |

### Поле Matrix у пользователя (`bot_users.room`)

В форме пользователя в колонку Matrix можно ввести: **логин** (если в настройках задан домен бота — он допишется как `@логин:домен`), полный **MXID** (`@user:server`) для личных уведомлений в DM, либо полный **id комнаты** (`!room_id:server`), если уведомления должны идти в явную комнату. Тестовое сообщение из панели для MXID с `@` открывает или использует DM с ботом; для `!…` — отправка в указанную комнату (бот должен быть участником).

### Когда нужен перезапуск бота

Бот при старте загружает из PostgreSQL секреты интеграции, пользователей, маршруты, справочники и `cycle_settings`.

**Фоновая подгрузка (hot reload):** каждые ~45 секунд (переменные `BOT_HOT_RELOAD`, `BOT_HOT_RELOAD_INTERVAL_SEC`, по умолчанию включено) бот перечитывает из БД пользователей, группы, маршруты, справочники и настройки цикла/шаблонов и обновляет интервалы опроса и расписание утреннего отчёта **без рестарта** процесса. Метки времени в **логах** процесса при смене таймзоны сервиса обновляются после **рестарта** бота и админки (форматтер привязан к старту процесса).

**Таймзона сервиса** (`/onboarding`, `BOT_TIMEZONE`): используется для расписания бота, времени в тестовых сообщениях пользователя и группы из панели и для `%(asctime)s` в логах бота и админки после перезапуска.

Отключить подгрузку: `BOT_HOT_RELOAD=0` (только рестарт бота подхватывает изменения в БД).

**Перезапуск процесса бота** по-прежнему нужен после:

- первичного ввода или смены **секретов** Redmine/Matrix в `app_secrets` (после стартового ожидания БД или смены URL/ключей вне hot reload);
- смены переменных только в **`.env`** (пути логов, `ADMIN_URL`, `APP_MASTER_KEY`, и т.д.);
- смены **Matrix `device_id`**, если клиент уже создал сессию с другим id — надёжнее рестарт.

**Админ-панель** от изменений конфигурации бота не перезапускается.

## 4. Смена пароля админки

Самообслуживания по форме «забыли пароль» нет. Варианты:

1. Другой администратор — в разделе **«Аккаунты панели»**.
2. Скрипт:
   ```bash
   python scripts/reset_admin_password.py --login admin --password 'NewPassword123'
   ```
3. Страница `/reset-password?token=…` — смена по одноразовому токену (таблица `password_reset_tokens`).

## 5. События и аудит

На странице **События** (`/events`) отображается журнал событий — записи бота и действия в панели (строки с меткой `[ADMIN]`).

**Что попадает в журнал:**
- Циклы опроса Redmine, отправка уведомлений, ошибки бота
- Вход / выход из панели
- Управление ботом (Старт / Стоп / Рестарт)
- CRUD-операции (при включённом `ADMIN_EVENTS_LOG_CRUD=1`)

**Фильтрация:** по диапазону дат. **Экспорт:** кнопка CSV.

Подробнее о каналах логирования и аудите: [AUDIT_LOGGING.md](AUDIT_LOGGING.md).

## 6. Типичные проблемы

### `password authentication failed for user "bot"`

Том `postgres_data` создан раньше с другим `POSTGRES_PASSWORD`. Варианты:

- Подставить в `DATABASE_URL` значение из текущего `.env`.
- Выровнять пароль:
  ```bash
  docker compose exec postgres psql -U bot -d via \
    -c "ALTER USER bot WITH PASSWORD 'новый_пароль';"
  ```
- Для чистого окружения: `docker compose down -v` (данные БД удалятся).

### `ERR_CONNECTION_REFUSED` на `127.0.0.1`

Админка слушает порт **8080** (`ADMIN_PORT` в `.env`). Полный URL: `http://127.0.0.1:8080/login`.

Если не работает:
```bash
docker compose ps
docker compose logs --tail=80 admin
```

### Бот спамит старыми уведомлениями после перезапуска

Очистите state:
```bash
docker compose exec postgres psql -U bot -d via \
  -c "DELETE FROM bot_issue_state;"
docker compose restart bot
```

## 7. Остановка и бэкап

```bash
docker compose down
```

Данные БД в томе `postgres_data` при `down` **не удаляются**. Бэкап:

```bash
docker compose exec postgres sh -lc \
  'pg_dump -U "$POSTGRES_USER" -d "$POSTGRES_DB"' > backup_$(date +%F).sql
```

Аварийный откат: [rollback-runbook.md](rollback-runbook.md).
