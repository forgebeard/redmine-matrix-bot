**Статус:** реализовано в кодовой базе Via (эпик 2026-04)

## Зафиксированные продуктовые решения

- Одна карточка **утренний отчёт**: расписание и шаблон `tpl_daily_report` вместе.
- **Убрать emoji везде** в текстах уведомлений и дефолтных шаблонах (включая метаданные бота `NOTIFICATION_TYPES` и справочник в БД — миграция `0023_clear_nt_emoji`).
- Убрать строку «Переменные: …» из конструктора; справочник — [notification_template_variables.md](notification_template_variables.md).
- **Вариант B** для режимов «Код» / «Блоки»: улучшение **серверной** декомпозиции (`prepare_body_html_for_decompose`, `/decompose-body`), а не только клиентских эвристик. **Вариант A** — доработка только `switchToBlocks` в браузере — отклонён.
- Превью: **пузырь в стиле Matrix** (тёмная подложка, токены `--bg-input` / `--fg` / `--accent-200` для ссылок), не пиксель-перфект под каждый клиент.

## Порядок работ (PR)

1. Emoji + миграция `notification_types`.  
2. Документация переменных + ссылка в [ADMINISTRATOR_GUIDE.md](ADMINISTRATOR_GUIDE.md).  
3. UI: скрыть переменные, ссылка на справку во вкладке.  
4. Карточка утреннего отчёта (зависимость от наличия `tpl_daily_report` в API; иначе заглушка).  
5. Декомпозиция + тесты round-trip.  
6. Matrix-превью.  
7. Токены радиуса в `template_block_editor.css`.

**Вне скоупа:** очистка emoji в переопределённых строках `notification_templates` без отдельного прод-решения.

## Минимум по сохранению

Две операции: «Сохранить расписание» и сохранение шаблона в редакторе (отдельные запросы).

## Зависимость PR4

Карточка с редактором `tpl_daily_report` требует записи шаблона в API (задача реестра `tpl-daily-registry` в плане Daily report). Если шаблона нет — показывается заглушка.
