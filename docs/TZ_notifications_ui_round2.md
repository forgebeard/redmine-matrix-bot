# ТЗ: UI вкладки «Уведомления» и редактора шаблонов (раунд 2)

> Obsolete: исторический документ эпохи block-editor. Актуальный контракт — code-only редактор и `tpl_test_message`/`tpl_*` в текущем README и профильных docs.

**Версия:** 2.10  
**Статус:** реализовано в коде (см. коммиты после 2026-04)

## 1. Цель

Вкладка **«Уведомления»** и `BlockEditor` визуально и по UX совпадают с остальной админкой: без инлайн-стилей в HTML, без технической подписи `emoji:`, превью Matrix не «отрывается» от темы карточек, компоновка утреннего отчёта читабельна.

**Зафиксировано:** включение/выключение **утреннего отчёта** сохраняется (стенд, отладка, пауза без деплоя); оформление — **переключатель в стиле `.switch`**, как у пользователей/групп.

## 3. Требования

| Тема | Решение |
|------|---------|
| Инлайны в onboarding | Классы в `settings.css`, префикс `daily-report__*`, `notifications-*` |
| Вкл/выкл утреннего отчёта | Единственный switch в строке заголовка карточки `Утренний отчёт` (`#daily_report_enabled`, без текстовой подписи рядом) |
| Справка по переменным | На вкладке «Уведомления» не показывается; не занимает место в интерфейсе |
| Превью Matrix | Фон/граница/радиус как у `.service-bubble`; без подписи «Имитация сообщения…»; фиксированный ритм пузыря (`max-width`/`min-height`) |
| Подпись `emoji:` | Поле `label` в `SettingDef` + JSON; в UI — человекочитаемый текст |
| Вкл/выкл шаблонов | Для каждого шаблона switch находится только в header-строке (`заголовок слева, switch справа`) в том же паттерне, что у `Утренний отчёт`; у отдельных блоков switch не показываются |
| Daily report layout | Две зоны: сначала «Шаблон сообщения», затем «Расписание» |
| Daily report time | Одно поле «Во сколько» (`HH:MM`) вместо двух отдельных полей «Час/Минута» |
| Карточки task-templates | `Новая задача`/`Изменение`/`Напоминание` рендерятся отдельными карточками уровня `service-bubble` (не вложенными `.field`) |
| Инлайны в JS | Классы `tpl-v2-*`, `is-hidden` вместо `element.style` где возможно |

## 4. Критерии приёмки

- В `#tab-notifications` нет атрибутов `style=` в статическом HTML.
- На вкладке «Уведомления» отсутствует ссылка «Справка по полям шаблонов Matrix».
- Рамка `.block-editor__preview` использует те же токены, что `.service-bubble` (`surface`, `border`, `radius`).
- В карточке «Утренний отчёт» шаблон и расписание визуально разделены на две подзоны и идут в порядке: шаблон → расписание.
- В header карточки `Утренний отчёт` есть только один switch, а внутри расписания нет дублирующего включателя.
- В BlockEditor нет кнопки «Расширенные настройки».
- Для каждого шаблона (`Новая задача`, `Изменение`, `Напоминание`) switch расположен в верхней header-строке рядом с заголовком.
- Header-строки `Новая задача`/`Изменение`/`Напоминание` рендерятся по тому же DOM/CSS-паттерну, что и `Утренний отчёт` (1:1 по структуре и выравниванию).
- Task-templates рендерятся отдельными карточками уровня `service-bubble`; header и switch находятся на том же структурном уровне, что и у `Утренний отчёт`.
- Внутри `#tpl-v2-fields` карточки шаблонов идут отдельным вертикальным стеком с единым ритмом отступов.
- У switch шаблонов нет текстов «Активен: …».
- Внутри тела редактора отсутствуют строки-дубли вида `Название + switch`.
- У отдельных блоков (`Заголовок`, `Тема`, `Статус...`) switch отсутствуют.
- В расписании daily report используется одно поле времени `HH:MM`.
- В предпросмотре нет текста «Имитация сообщения в Matrix.», а внутренний пузырь не «гуляет» по размерам.
- `pytest tests/test_template_integrity.py` зелёный.
- Расписание и шаблоны сохраняются; `#daily_report_enabled` синхронизируется с API.
